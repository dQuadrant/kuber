# Release 4.1.0 (2026-03-12)

This release builds on `v4.0.0` with new features, bug fixes, devnet tooling improvements, and Docker/CI enhancements.

## Highlights
- New `cardano-submit-api` compatible endpoint in `kuber-server`.
- Docker image now auto-switches between `kuber-server` and `kuber-hydra` based on the `HYDRA_URL` environment variable.
- Hydra WebSocket connection is more robust with URL validation at startup and a configurable connect timeout.
- Devnet upgraded to Hydra 1.3.0.

## Changes

### New Features
- Added a standard `cardano-submit-api` compatible endpoint to `kuber-server` for broader tooling compatibility.
- Docker entrypoint now automatically selects `kuber-server` or `kuber-hydra` based on the presence of the `HYDRA_URL` environment variable.
- Added WebSocket connect timeout for the Hydra node connection to fail fast on misconfigured URLs.
- Added Docker image CI pipeline to build and publish images for both `kuber` and `kuber-hydra` on tag push.

### Fixes
- Fixed transaction fee starting at a non-zero value for already-balanced Hydra transactions, which caused insufficient input errors.
- Fixed `mintResult` response missing `hash` and `cborHex` fields in native token minting.
- Fixed native token asset name with underscores causing a minting error.
- Removed stale reference to missing `KuberHydraService`.

### Hydra WebSocket Improvements
- Hydra connection URL is now parsed and validated at startup rather than on first use.
- Strengthened URL verification to surface misconfiguration errors earlier.

### Devnet Tooling
- Upgraded devnet to **Hydra 1.3.0**.
- Added a `reset-cluster.sh` convenience script for resetting the local devnet cluster (including volumes).
- Added Cardano node config and genesis files for the devnet cluster.
- Updated devnet docker-compose peer addresses and reduced contestation period to 10 s.
- Improved credential generation, UTxO verification, and seeding scripts.

### Docs
- Added a full end-to-end example guide covering devnet setup to transaction submission.
- Updated docs to support **kuber-client v3.4.0**.
- Added Hydra JS client guides for local devnet and testnet/mainnet.
- Upgraded Node.js versions referenced in documentation.
- Refined getting started, installation, and Hydra API reference sections.

## Compatibility
- cardano-api: 10.1
- kuber-client: v3.4.0

## Tested with
